# 验收标准

## 1. 功能验收

### 1.1 资源适配配置

#### AC-1.1: 资源配置 CRUD
**Given** 系统管理员已登录
**When** 访问资源配置页面
**Then** 可以创建、查看、更新、删除资源配置

**验收标准：**
- [ ] 可以创建低/中/高三种级别的配置
- [ ] 配置参数可以自定义（数据库连接、模型选择等）
- [ ] 配置验证生效（必填项、格式检查）
- [ ] 默认配置自动创建
- [ ] 配置删除时检查是否被使用

#### AC-1.2: 配置应用
**Given** 已存在资源配置
**When** 应用配置到系统或知识库
**Then** 配置生效，新知识库使用新配置

**验收标准：**
- [ ] 系统级配置变更影响新知识库
- [ ] 知识库级配置覆盖系统默认
- [ ] 配置变更后提示是否需要重建索引
- [ ] 重建任务异步执行

#### AC-1.3: 配置测试连接
**Given** 已填写配置参数
**When** 点击测试连接按钮
**Then** 显示连接测试结果

**验收标准：**
- [ ] 成功时显示延迟和版本信息
- [ ] 失败时显示具体错误原因
- [ ] 支持测试多种后端类型

---

### 1.2 数据存储适配器

#### AC-2.1: SQLite 适配器
**Given** 系统配置为低资源级别
**When** 创建知识库并上传文档
**Then** 文档存储在 SQLite，支持检索

**验收标准：**
- [ ] 支持文档 CRUD
- [ ] 支持全文检索（FTS5）
- [ ] 查询延迟 < 500ms（1000 文档）
- [ ] 单文件存储，零配置

#### AC-2.2: PostgreSQL + pgvector 适配器
**Given** 系统配置为中资源级别
**When** 创建知识库并上传文档
**Then** 文档存储在 PostgreSQL，支持向量检索

**验收标准：**
- [ ] 支持文档 CRUD
- [ ] 支持向量检索（HNSW 索引）
- [ ] 支持全文检索（tsvector）
- [ ] 支持混合检索（应用层 RRF）
- [ ] 查询延迟 < 200ms（10万 文档）

#### AC-2.3: Elasticsearch 适配器
**Given** 系统配置为高资源级别
**When** 创建知识库并上传文档
**Then** 文档存储在 ES，支持混合检索

**验收标准：**
- [ ] 支持文档 CRUD
- [ ] 支持原生混合检索（RRF）
- [ ] 支持集群模式
- [ ] 查询延迟 < 100ms（100万 文档）

---

### 1.3 增强检索引擎

#### AC-3.1: 多路召回
**Given** 系统配置为混合检索
**When** 执行查询
**Then** 并行执行多种检索方式

**验收标准：**
- [ ] 向量检索和关键词检索并行执行
- [ ] 检索结果去重
- [ ] 支持配置检索方式权重

#### AC-3.2: 融合策略
**Given** 多路召回结果
**When** 应用融合策略
**Then** 返回融合后的排序结果

**验收标准：**
- [ ] RRF 融合结果正确
- [ ] 加权融合结果正确
- [ ] 融合后召回率 > 85%

#### AC-3.3: 重排序
**Given** 初始检索结果
**When** 启用重排序
**Then** 返回重排序后的结果

**验收标准：**
- [ ] 支持 BGE Reranker
- [ ] 重排序提升准确率 > 10%
- [ ] 重排序延迟 < 100ms

**Given** 10万 个文档
**When** 构建索引

**Given** 100万 个文档

## 5. 验收流程

### 5.1 验收步骤

1. **准备环境**
   - 部署测试环境
   - 准备测试数据
   - 配置监控

2. **执行测试**
   - 功能测试
   - 性能测试
   - 安全测试

3. **记录结果**
   - 记录测试数据
   - 截图/录屏
   - 问题跟踪

4. **评审会议**
   - 展示测试结果
   - 讨论问题
   - 决定是否通过

5. **签署验收**
   - 验收报告
   - 签字确认
   - 归档保存
